Do's and Don'ts

Do

  • Use direct, action-oriented copy: “Order placed. Inventory updated. Warehouse has their pick list.”
  • Show empty states that guide the user: “No picks waiting. Once a salesman places an order, it'll show up here.”
  • Maintain 44px minimum touch targets on all buttons and interactive controls.
  • Use font-weight and letter-spacing for hierarchy before reaching for color or size.
  • Support light and dark mode as equals — every color token has a dark counterpart.
  • Support RTL layout for Arabic localization via [dir="rtl"] attribute.
  • Give every interactive element a visible focus-ring outline — not just inputs.
  • Reserve on-surface-faint for decorative use only; use on-surface-subtle or darker for real copy.
  • Treat sidebar vs. top bar as a display preference over the same route tree — never fork nav data per mode.

Don't

  • Use ERP jargon in UI copy. No “Error code: ERR-8472,” no “streamlined onboarding experience.”
  • Hide critical data (stock levels, order status, pricing) behind hover or expand interactions.
  • Add playful elements — Nova is a tool for daily work, not a consumer app.
  • Use fully sharp (0px) corners on any interactive element.
  • Add animation beyond 0.15–0.25s transitions — no bounce, no spring, no parallax.
  • Introduce a grid framework or CSS framework — the existing layout patterns are sufficient.
  • Mirror icons whose meaning isn't directional (refresh, play, checkmarks, logos) in RTL.
  • Use opacity alone to show "disabled" — use the explicit disabled-bg/disabled-text pair.
